# 技术架构

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 语言 | Python 3.11+ | 全部代码 |
| UI | Streamlit | Web 界面（5 页 SPA） |
| 数据库 | SQLite | 本地数据存储 |
| 浏览器 | Playwright (Chromium) | 反反爬，JS 渲染页面抓取 |
| HTTP | requests | 简单 API 调用（Taobao Suggest） |
| 图表 | Plotly | 雷达图、热力图 |
| 图标 | Material Symbols | UI 图标字体 |
| NFC | TF-IDF + 余弦相似度 | 跨平台商品标题匹配 |

## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    Streamlit UI (5 页)                   │
│  首页 │ 雷达 │ 看板 │ 工具 │ 历史                        │
├─────────────────────────────────────────────────────────┤
│  pages/  ── 页面渲染                                     │
│  src/    ── 业务逻辑                                     │
│    ├─ config.py       配置加载                            │
│    ├─ database.py     SQLite CRUD                        │
│    └─ modules/        ┐                                  │
│       ├─ discovery.py  品类发现（Taobao Suggest API）     │
│       ├─ matcher.py    标题匹配（TF-IDF）                 │
│       ├─ arbitrage.py  价差计算（净利润/ROI）             │
│       ├─ pusher.py     推送（QQ邮件/Server酱）            │
│       ├─ reporter.py   日报生成                           │
│       └─ scrapers/     三平台抓取器                       │
│           ├─ playwright_base.py  Playwright 基类         │

├─────────────────────────────────────────────────────────┤
│  SQLite (data/ai_storekeeper.db)                        │
│  9 张表: products · price_history · opportunities ·     │
│  user_blacklist · system_config · workflow_runs ·        │

```

## 数据流

```
种子品类(30个)
  → Taobao Suggest API 扩展
  → keyword_pool(~400个关键词)
  → 每轮取 20 个未扫描
  → 3 个 Playwright 抓取器并行抓取
  → TF-IDF 跨平台标题匹配
  → 净利润/ROI 计算
  → opportunities 表
  → ROI>30% 实时推送邮件
  → 看板人工审核
  → 9:00/21:00 日报
```

## 关键设计决策

| 决策 | 理由 |
|------|------|
| SQLite 而非 MySQL | 本地单机，零部署成本 |
| Playwright 而非 requests | 1688/拼多多/闲鱼均为 JS 渲染 SPA |
| TF-IDF 而非 LLM | 零成本、离线可运行 |
| Taobao Suggest API | 免费、无需 API Key |
| QQ 邮箱 SMTP | 免费、国内用户首选 |
| Streamlit | 快速原型、无需前端框架 |

# 代码规范

## Python 版本

Python 3.11+

## 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 模块/文件名 | snake_case | `scraper_1688.py` |
| 类名 | PascalCase | `Scraper1688` |
| 函数/方法 | snake_case | `get_stats()` |
| 变量 | snake_case | `total_opportunities` |
| 常量 | UPPER_SNAKE | `PLATFORM_NAME` |
| 私有方法 | `_` 前缀 | `_parse_price()` |

## 文件组织

```
ai_storekeeper/
├── app.py                  # Streamlit 入口
├── pages/                  # UI 页面（每页一个 render() 函数）
├── src/                    # 后端逻辑（不依赖 Streamlit）
│   ├── config.py           # 配置加载
│   ├── database.py         # 数据层
│   └── modules/            # 业务模块
│       ├── discovery.py    # 品类发现
│       ├── matcher.py      # 标题匹配
│       ├── arbitrage.py    # 价差计算
│       ├── pusher.py       # 推送
│       ├── reporter.py     # 日报
│       └── scrapers/       # 抓取器
└── tests/                  # 测试（mirror src/ 结构）
```

## 文档注释

- **不写多余注释**（代码即文档）
- 只在以下情况写注释：
  - 复杂算法说明
  - 临时 workaround 需要解释原因
  - 公开 API 的 docstring

## 类型标注

- 支持但不强制
- 公开函数建议标注返回值类型

## 测试规范

- 测试文件：`tests/test_*.py`
- 每个模块至少 3 个测试
- 测试使用真实 SQLite（不 mock）
- 测试覆盖率目标：核心逻辑 80%+

## import 顺序

1. 标准库
2. 第三方库
3. 本地模块

每组之间空一行。

## Config 键名规范

全部小写 + 下划线：
```
[scanner]
seed_categories = ...
keywords_per_round = 20
scan_interval_minutes = 60
```

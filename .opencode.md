## 工作规则

### 每次工作开始前
1. 检查 `devlog/` 最新一条日志，了解上下文中断点
2. 检查 `docs/05-开发计划.md` 当前待办阶段

### 每次工作完成后
1. 在 `devlog/` 创建当日日志（如已存在则更新）
2. 记录：今天完成了什么、待办还剩什么、遇到了什么问题
3. 更新 `docs/05-开发计划.md` 中对应任务的勾选状态

### 代码修改原则
- 读文件 → 理解上下文 → 编辑
- 不随意重构，除非用户明确要求
- 遵循 `docs/06-代码规范.md` 的命名和格式
- 修改完后跑测试确认不破坏现有功能

### 测试命令
```bash
cd ai_storekeeper
python -m pytest tests/ -v
```

### 启动命令
```bash
cd ai_storekeeper
streamlit run app.py
```

## 重要注意事项

- **用户是技术小白**，解释时用最简单的语言，不要用术语
- **Google Fonts 在大陆可能被屏蔽**，Material Symbols 图标可能显示为乱码。替代方案：国内 CDN 镜像
- **爬虫无登录态时返回 demo 数据**（合成数据）供 UI 演示
- **QQ 邮箱 SMTP 已配置**（465/SSL），不要随意修改
- **config.ini 中的 smtp_pass 是授权码**，不要泄露
- **不要主动提交 git**，除非用户明确要求

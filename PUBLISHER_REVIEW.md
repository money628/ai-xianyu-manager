# AI店长 一键铺货 现状与优化讨论

## 当前实现状态

代码位置: `src/modules/publisher.py`

### 已实现的防风控措施（10层）

| 层级 | 技术 | 说明 |
|------|------|------|
| 1 | playwright-stealth | 隐藏 navigator.webdriver、chrome.runtime |
| 2 | Chrome 启动参数 | --disable-blink-features=AutomationControlled |
| 3 | WebRTC 泄露防护 | disable_non_proxied_udp |
| 4 | 移动端模拟 | iPhone 17 + iOS Safari UA + 3x retina + touch |
| 5 | 地理一致性 | 杭州经纬度 + Asia/Shanghai 时区 |
| 6 | 硬件伪装 | 6核CPU + 4GB RAM + 5个plugins |
| 7 | Battery API 伪造 | 满电充电状态 |
| 8 | 权限 API 拦截 | 拦截通知权限查询 |
| 9 | 人类行为模拟 | 打字40-150ms/字、字段间随机停顿、偶尔删除重打、随机鼠标移动 |
| 10 | 描述话术 | "个人闲置""自用""换新" |

### 未验证的风险点

1. **从未真实验证过** — 代码写好了但没有实际跑过一次铺货
2. **闲鱼表单 DOM 可能已变** — input/textarea 选择器是硬编码的
3. **不点击发布按钮** — 只填表单，留给人手动检查发布
4. **没有错误恢复** — 如果某个字段没找到，静默跳过
5. **图片上传未测试** — 商品图片可能不存在或 URL 无效

### 当前流程

```
用户点击"🚀 自动填写闲鱼发布"
  → subprocess 启动独立 Python 脚本
  → 打开 Playwright + Chrome 浏览器
  → 加载闲鱼登录态
  → 导航到 h5.m.goofish.com/publish/edit
  → 自动填写: 标题、价格、描述
  → 尝试上传图片
  → 浏览器保持打开，用户手动检查 + 发布
```

### 风控重点（来自 fingerprint.com 研究）

Xianyu/淘宝系主要检测:
1. **Canvas 指纹** — GPU 渲染差异
2. **WebGL 指纹** — 3D 图形渲染
3. **字体列表** — 安装了哪些字体
4. **AudioContext 指纹** — 音频振荡器差异
5. **TLS 握手** — 加密套件组合
6. **行为模式** — 鼠标轨迹、打字节奏、页面停留时间
7. **IP/设备关联** — 同 IP 多账号、同设备多次操作

## 可讨论的优化方向

### 方向A: 补齐检测覆盖
- 添加 AudioContext 指纹伪造
- 随机化字体列表
- 鼠标轨迹用贝塞尔曲线
- 模拟"犹豫"行为（在发布按钮附近停留、滚动检查）

### 方向B: 降低操作频率
- 硬限制每天最多铺货 N 次
- 两次铺货间隔最少 30 分钟
- 记录操作日志供复查

### 方向C: 表单选择器自适应
- 不用硬编码选择器，多策略 fallback
- 检测页面结构变化，失败时截图+提示用户手动操作

### 方向D: 不自动化发布
- 只把信息复制到剪贴板
- 在 Streamlit 页面显示铺货清单
- 用户手动去闲鱼发布（最安全）

### 方向E: 多账号轮换
- 闲鱼也搞多账号
- 每个号每天只发 1-2 件

---

## 核心问题

**是否值得冒封号风险？**

一键铺货收益: 省 2 分钟填表
封号代价: 闲鱼号废了，重新注册+养号需要时间

建议: 优先验证匹配效果（找到更多套利机会），铺货先用手动或半自动方式。

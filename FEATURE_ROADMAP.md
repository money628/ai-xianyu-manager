## 一、项目概况

**AI店长** 是一个本地 Windows 运行的跨平台价差套利工具。

**核心链路**：关键词池 → PDD API(真实) + 闲鱼 Playwright(真实) → 双向交叉匹配 → ROI 计算 → SQLite 入库 → QQ邮箱/钉钉推送

**技术栈**：Python + Streamlit + Playwright + SQLite + PDD开放平台API + 钉钉Stream SDK

## 三、当前瓶颈与风险

| 问题 | 严重度 | 说明 |
|------|--------|------|
| PDD API 每日限额 | 🔴 高 | 2000 次/天用完就没数据，匹配率直接归零 |
| E2E 匹配率未验证 | 🔴 高 | PDD 限流期间无法实测真实匹配效果 |
| 闲鱼防风控 | 🟡 中 | Playwright 自动化可能被检测封号 |
| 单机运行 | 🟡 中 | 电脑关机服务就停了 |
| 1688 未攻克 | 🟢 低 | 少一个采购数据源 |

## 五、推荐路线（3 选 1）

### 路线 A：稳健增长（推荐）
```
多PDD账号轮换 → 实机验证匹配 → 淘宝接入 → Web部署
```
适合：现在就想出成果，稳定增加套利机会数量。

### 路线 B：AI 驱动
```
AI标题生成 → AI选品推荐 → AI风控 → Web部署
```
适合：想做差异化智能化工具，未来可付费。

### 路线 C：全自动工厂
```
Docker化 → Web部署 → 自动更新 → 多用户 → 付费
```
适合：想做成 SaaS 产品服务他人。
